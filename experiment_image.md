# 视觉短视频主观评分实验设计

## 1. 实验目的

本实验用于采集被试观看短视频过程中的主观情绪与体验评分，并在同一 session 内**同步连续采集 EEG 数据**。行为任务各阶段通过事件标记与 EEG 时间轴对齐，便于后续联合分析。每个 trial 呈现一个短视频，视频结束后被试完成 6 个维度的 5 点量表评分。

## 2. 实验结构

实验程序包含三个主要阶段：

1. 实验说明首页
2. 练习 trial
3. 正式实验

正式实验共 540 个 trial。后续正式实验将分为 6 个 block，每个 block 包含 90 个视频。

行为评分任务与 EEG 采集在同一 session 内并行进行：被试完成观看与评分的同时，程序持续记录脑电信号，并在各实验阶段边界发送同步事件标记。

## 3. EEG 同步采集

### 3.1 采集时机

- 正式实验开始前启动 EEG 连续采集，全部 trial 结束后停止采集。
- 练习 trial 是否同步采集 EEG 由实验配置决定；建议与正式 trial 保持一致，以便流程统一。
- 可选：正式实验前采集一段静息基线 EEG（`baseline_start` / `baseline_end`），供后续预处理参考。

### 3.2 同步方式

- 程序在每个 trial 及各阶段边界发送事件标记（marker/trigger），写入 EEG 事件流或与 trigger 硬件同步。
- 行为数据记录各阶段的 onset/offset 时间戳；`events.json` 与 EEG 连续数据通过事件代码和时间戳对齐。
- 同一被试、同一 session 的行为评分、播放列表与 EEG 数据保存在对应目录下，通过 `subject_id`、`session_id` 和时间戳关联。

### 3.3 事件标记

| 事件代码 | 事件名称 | 含义 |
| --- | --- | --- |
| 101 | `session_start` | 实验 session 开始 |
| 102 | `session_end` | 实验 session 结束 |
| 110 | `baseline_start` | 静息基线开始 |
| 111 | `baseline_end` | 静息基线结束 |
| 120 | `block_start` | block 开始 |
| 121 | `block_end` | block 结束 |
| 130 | `fixation_on` | 注视十字出现 |
| 131 | `fixation_off` | 注视十字结束 |
| 132 | `video_on` | 视频开始播放 |
| 133 | `video_off` | 视频播放结束 |
| 134 | `blank_on` | 空屏开始 |
| 135 | `blank_off` | 空屏结束 |
| 136 | `rating_on` | 行为评分界面出现 |

### 3.4 数据输出

EEG 与行为数据分别保存，典型目录结构如下：

```
records_storage/
  {subject_id}/
    session_{session_id}/
      {timestamp}/
        events.json            # 事件标记与时间戳
        metadata.json          # session 元数据
        behavioral_ratings.csv # 行为评分
        # EEG 连续数据（格式依采集设备而定）
```

## 4. 单 Trial 流程

每个正式 trial 按以下顺序执行：

| 阶段 | 时长 | 说明 |
| --- | --- | --- |
| 注视点 | 1 s | 屏幕中央呈现注视点，提示被试保持注视并准备观看视频。 |
| 视频播放 | 按视频长度或默认播放时长 | 播放当前 trial 的视频刺激；需要设置按键跳过功能，允许在必要时提前结束当前视频。 |
| 空屏过渡 | 1 s | 视频结束后呈现空屏，作为视频刺激与评分阶段之间的过渡。 |
| 主观评分 | 15-30 s | 被试完成 6 个评分题目。正式实验中评分页限时，超时后按当前已选择的评分记录并进入下一阶段。 |
| ITI | 1 s | trial 间隔，用于进入下一个 trial 前的短暂休息。 |

练习 trial 与正式 trial 使用相同流程，但评分页不限时，以便被试熟悉评分题目和操作方式。

## 5. 实验说明首页

实验说明首页需要向被试说明：

- 本任务需要观看一系列短视频，并在每个视频后根据观看时的真实感受完成主观评分。
- 评分没有正确或错误答案，请根据即时主观体验作答。
- 每个题目采用 1-5 分量表。
- 正式实验评分阶段有时间限制，请在页面出现后尽快作答。
- 视频播放阶段支持按键跳过；如无特殊情况，应尽量完整观看视频。
- 练习 trial 用于熟悉流程，练习评分不限时。
- 实验期间将同步采集 EEG，被试需佩戴脑电采集设备并尽量保持身体稳定。

## 6. 评分题目

每个视频后呈现 6 个评分维度：

1. Valence：愉悦度
2. Arousal：唤醒度
3. Immersion：沉浸感
4. Interest：兴趣度
5. Visual：视觉吸引力
6. Auditory：听觉愉悦度

所有维度均使用 5 点 Likert 量表，1 表示该维度的低端，5 表示该维度的高端。

### 6.1 Valence / 愉悦度

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very negative / 非常消极 | 观看视频时感受到非常消极的情绪，例如悲伤、愤怒或厌恶。 |
| 2 | Negative / 消极 | 观看视频时感受到一定消极情绪，但强度低于 1 分。 |
| 3 | Neutral / 中性 | 观看视频时没有明显的积极或消极情绪。 |
| 4 | Positive / 积极 | 观看视频时感受到一定积极情绪，但强度低于 5 分。 |
| 5 | Very positive / 非常积极 | 观看视频时感受到非常积极的情绪，例如快乐、兴奋或愉悦。 |

### 6.2 Arousal / 唤醒度

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very calm / 非常平静 | 观看视频时感到非常平静，心率和呼吸缓慢且稳定。 |
| 2 | Calm / 平静 | 观看视频时感到较为平静，但程度低于 1 分。 |
| 3 | Neutral / 中性 | 心率和呼吸略有升高，但没有明显兴奋或焦虑。 |
| 4 | Excited / 兴奋 | 心率和呼吸明显升高，观看视频时感到兴奋和被刺激。 |
| 5 | Very excited / 非常兴奋 | 心率和呼吸极度升高，观看视频时感到高度唤醒和强烈刺激。 |

### 6.3 Immersion / 沉浸感

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very disconnected / 非常疏离 | 与视频内容和画面事件完全缺乏连接感。 |
| 2 | Disconnected / 疏离 | 与视频内容有一定疏离感，但程度低于 1 分。 |
| 3 | Neutral / 中性 | 对视频内容和画面事件有中等程度的投入。 |
| 4 | Engaged / 投入 | 对视频内容和画面事件高度投入。 |
| 5 | Very engaged / 非常沉浸 | 完全沉浸在视频内容和画面事件中。 |

### 6.4 Interest / 兴趣度

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very uninterested / 非常不感兴趣 | 认为视频极其无趣、乏味。 |
| 2 | Uninterested / 不感兴趣 | 认为视频有些无趣，但程度低于 1 分。 |
| 3 | Neutral / 中性 | 认为视频有中等程度的趣味性。 |
| 4 | Interested / 感兴趣 | 认为视频很有趣、很吸引人。 |
| 5 | Very interested / 非常感兴趣 | 认为视频极其有趣、非常吸引人。 |

### 6.5 Visual / 视觉吸引力

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very unappealing / 非常不吸引 | 认为视频的视觉方面极其不吸引人，观看体验不佳。 |
| 2 | Unappealing / 不吸引 | 认为视频的视觉方面有些不吸引人，但程度低于 1 分。 |
| 3 | Neutral / 中性 | 认为视频的视觉方面具有中等吸引力。 |
| 4 | Appealing / 吸引 | 认为视频的视觉方面很吸引人，观看体验较好。 |
| 5 | Very appealing / 非常吸引 | 认为视频的视觉方面极其吸引人，观看体验非常好。 |

### 6.6 Auditory / 听觉愉悦度

| 分值 | 含义 | 解释 |
| --- | --- | --- |
| 1 | Very unpleasant / 非常不愉悦 | 认为视频的声音极其不愉悦，听觉体验很差。 |
| 2 | Unpleasant / 不愉悦 | 认为视频的声音有些不愉悦，但程度低于 1 分。 |
| 3 | Neutral / 中性 | 认为视频的声音具有中等程度的吸引力。 |
| 4 | Pleasant / 愉悦 | 认为视频的声音较有吸引力，听觉体验较好。 |
| 5 | Very pleasant / 非常愉悦 | 认为视频的声音极具吸引力，听觉体验非常好。 |

## 7. 视频材料与播放列表

视频从 `visual-video-task/video_library` 中读取。每个视频需要在视频库清单中记录以下信息：

- `id`：视频唯一编号。
- `file`：视频文件相对路径。
- `content_dimension`：视频所属内容维度，共 10 个内容维度。
- `duration_sec`：可选，视频时长；无法读取真实时长时使用默认播放时长。

建议 `manifest.json` 中的单个视频条目格式如下：

```json
{
  "id": "stim_001",
  "file": "stim_001.mp4",
  "content_dimension": "dimension_01",
  "duration_sec": 8.0
}
```

## 8. 随机化与平衡规则

每次实验开始时根据 `video_library` 随机生成一次 `videoplaylist`，该播放列表在当前被试当前 session 内保持固定，并随行为数据一起保存，便于复现实验顺序。

随机化需要满足以下规则：

- 正式实验共 540 个 trial，包含 10 个内容维度。
- 整个 session 内需要平衡内容维度：每个内容维度应出现 54 个视频。
- 后续分 block 后，每个 block 包含 90 个视频。
- 每个 block 内也需要平衡内容维度：每个内容维度应出现 9 个视频。
- 在满足内容维度平衡的前提下，视频顺序随机排列。
- 同一 block 内应避免同一内容维度连续出现过多次；如无法完全避免，应至少记录该随机序列并允许后续检查。
- 若某个内容维度可用视频数量不足，应在实验开始前报错，而不是在实验中降级为不平衡播放列表。

## 9. 正式实验流程

正式实验开始后执行以下流程：

1. 连接并校验 EEG 采集设备，启动连续采集并发送 `session_start` 标记。
2. （可选）采集静息基线 EEG，发送 `baseline_start` / `baseline_end` 标记。
3. 读取 `video_library` 并校验视频文件是否存在。
4. 校验视频内容维度是否完整，且每个视频只有一个明确的 `content_dimension`。
5. 根据 10 个内容维度生成平衡的 540-trial `videoplaylist`。
6. 如启用 block，将 540 个 trial 划分为 6 个 90-trial block，并保证每个 block 内 10 个内容维度平衡；每个 block 开始/结束时发送 `block_start` / `block_end` 标记。
7. 对每个 trial 执行：注视点、视频播放、空屏、评分、ITI；各阶段边界发送对应事件标记。
8. 每个 trial 保存视频编号、内容维度、block 编号、trial 编号、各阶段时间戳、跳过按键事件、评分结果和评分是否超时。
9. 实验结束后发送 `session_end` 标记，停止 EEG 采集，并保存完整行为评分表、事件日志、播放列表和 EEG 数据。

## 11. 实现注意事项

- 当前目标参数为：注视点 1 s、空屏 1 s、ITI 1 s、正式评分 15-30 s。
- 练习 trial 的评分页不限时，但其余阶段应与正式 trial 保持一致。
- 正式评分页需要在 15-30 s 范围内设置限时策略；如采用随机限时，需要记录每个 trial 的实际评分限时。
- 视频播放阶段需要支持按键跳过，并记录跳过事件与跳过时间。
- 评分维度应固定为 6 个题目，并使用 1-5 分量表。
- 视频库需要补充 `content_dimension` 元数据，否则无法实现内容维度平衡随机化。
- 当前 `video_library/manifest.json` 仅包含 10 个本地刺激示例；正式实验前需要扩展为可支持 540 个 trial 的完整视频库。
- 行为任务与 EEG 采集必须同步启动与结束；各 trial 阶段切换时须发送对应事件标记，保证后续可对齐分析。
- EEG 设备类型与连接参数在 `config.yaml` 中配置（如 `device_type`、`sfreq`）；正式采集前需完成阻抗检查与信号质量确认。

## 12. 程序首页实验说明

以下文字可用于实验程序首页，面向被试展示。

### 实验说明

欢迎参加本次短视频观看与主观评分实验。

在本实验中，你将佩戴脑电采集设备（电极帽），同时观看一系列短视频并完成主观评分。程序会同步记录你的脑电信号，用于科研分析。请在整个实验过程中尽量保持头部和身体稳定，减少说话、吞咽和大幅度动作。

每个视频结束后，请根据你在观看该视频时的真实感受，对视频进行 6 个维度的评分。评分没有正确或错误答案，请按照你的第一感受作答。

### 实验流程

实验开始后，每个 trial 会按照以下顺序进行：

1. 屏幕中央先出现一个注视点，请注视屏幕中央并做好观看准备。
2. 随后播放一个短视频。请尽量完整观看视频；如果需要提前结束当前视频，可以使用指定按键跳过。
3. 视频结束后会出现短暂空屏。
4. 空屏结束后进入评分页面，请根据刚刚观看视频时的感受完成 6 个评分题目。
5. 评分完成后，程序会短暂休息并自动进入下一个视频。

正式实验前会先进行一个练习 trial。练习 trial 的流程与正式实验相同，但评分页面不限时，目的是帮助你熟悉操作和评分方式。正式实验中，评分页面有时间限制，请在页面出现后尽快完成评分。

### 评分方式

每个题目使用 1-5 分评分。一般来说，1 表示该维度非常低或非常弱，3 表示中等或中性，5 表示该维度非常高或非常强。

你需要完成以下 6 个维度的评分：

1. 愉悦度 Valence

   请评价你观看视频时的情绪是偏消极还是偏积极。

   - 1 分：非常消极，例如感到悲伤、愤怒、厌恶或不舒服。
   - 3 分：中性，没有明显积极或消极情绪。
   - 5 分：非常积极，例如感到开心、兴奋、愉快或享受。

2. 唤醒度 Arousal

   请评价你观看视频时身体和情绪被激活的程度，也就是你有多平静或多兴奋。

   - 1 分：非常平静，心情和身体状态都很放松。
   - 3 分：中等，没有明显平静或兴奋。
   - 5 分：非常兴奋或被强烈刺激，感觉心跳、呼吸或情绪反应明显增强。

3. 沉浸感 Immersion

   请评价你是否投入到视频内容中，是否有被视频吸引进去的感觉。

   - 1 分：非常疏离，几乎没有投入到视频中。
   - 3 分：中等，有一定关注但没有明显沉浸。
   - 5 分：非常沉浸，感觉完全被视频内容吸引。

4. 兴趣度 Interest

   请评价你觉得这个视频是否有趣、是否愿意继续观看类似内容。

   - 1 分：非常无趣或乏味。
   - 3 分：中等，有一定趣味但不强烈。
   - 5 分：非常有趣、很吸引人。

5. 视觉吸引力 Visual

   请评价视频画面本身带给你的视觉体验，例如画面是否好看、清晰、舒服、有吸引力。

   - 1 分：视觉上非常不吸引人，观看体验较差。
   - 3 分：中等，视觉体验一般。
   - 5 分：视觉上非常吸引人，画面体验很好。

6. 听觉愉悦度 Auditory

   请评价视频声音带给你的听觉体验，例如声音、音乐或音效是否舒服、悦耳、有吸引力。

   - 1 分：声音非常不愉悦，听觉体验较差。
   - 3 分：中等，听觉体验一般。
   - 5 分：声音非常愉悦，听觉体验很好。

请注意，每个评分都只针对刚刚观看的那个视频。即使连续观看多个视频，也请独立评价当前视频，不需要和前面的视频进行比较。
