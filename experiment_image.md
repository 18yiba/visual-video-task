# 图片偏好对齐实验

## 1. 实验目的

本实验用于采集被试观看情绪图片时的主观偏好、情绪评分与同步 EEG 数据，为后续图片偏好对齐模型提供行为标签和脑电特征。行为任务各阶段通过事件标记与 EEG 时间轴对齐，便于后续进行单 trial 级别的联合分析。每个 trial 呈现一张情绪图片；首次呈现时被试完成愉悦度、唤醒度和视觉感受评分，重复呈现时主要用于提高 EEG 信噪比和模型泛化评估。

## 2. 实验结构

实验程序包含三个主要阶段：

1. 实验说明首页
2. 练习 trial
3. 正式实验

正式实验共 600 个 trial，由 120 张情绪图片重复呈现 5 次构成。120 张图片包含正性 40 张、中性 40 张、负性 40 张。正式实验分为 5 个 block，每个 block 包含 120 个 trial。

Block 1 为全量图片的首次观看与主观评分；Block 2-5 为同一批图片的乱序重复呈现，主要用于纯脑电降噪采集，并在约 10% trial 中加入简单注意力按键任务。

本实验将两种评分方式定义为两个图片范式，分别对应 `tasks` 目录下的两个任务文件；同一 session 只运行其中一种范式：


| 范式    | `task_mode` | 任务文件                    | 评分方式                                                                               |
| ----- | ----------- | ----------------------- | ---------------------------------------------------------------------------------- |
| 图片范式一 | `image_a`   | `tasks/task_image_a.py` | 三个 rating 题目在同一屏显示；被试用鼠标完成选择后，再通过鼠标点击进入下一屏。                                        |
| 图片范式二 | `image_b`   | `tasks/task_image_b.py` | 连续呈现 3 个 rating 题目；每题限时 2-3 s，超时即跳到下一屏，要求快速反应；操作方式仍为鼠标选择，后续可通过键盘 `F` / `J` 调节选项位置。 |


两种范式共用相同的刺激材料、Block 结构、EEG 同步规则和类型 B（纯脑电降噪）流程，差异仅在 Block 1 的评分交互方式。

行为评分任务与 EEG 采集在同一 session 内并行进行：被试完成观看、评分和注意力任务的同时，程序持续记录脑电信号，并在各实验阶段边界发送同步事件标记。

## 3. EEG 同步采集



### 3.1 采集时机

- 正式实验开始前启动 EEG 连续采集，全部 trial 结束后停止采集。
- 练习 trial 是否同步采集 EEG 由实验配置决定；建议与正式 trial 保持一致，以便流程统一。
- 可选：正式实验前采集一段静息基线 EEG（`baseline_start` / `baseline_end`），供后续预处理参考。



### 3.2 同步方式

- 程序在每个 trial 及各阶段边界发送事件标记（marker/trigger），写入 EEG 事件流或与 trigger 硬件同步。
- 行为数据记录各阶段的 onset/offset 时间戳；`events.json` 与 EEG 连续数据通过事件代码和时间戳对齐。
- 同一被试、同一 session 的行为评分、图片播放列表与 EEG 数据保存在对应目录下，通过 `subject_id`、`session_id` 和时间戳关联。
- 后续建模时按图片维度进行 8:2 泛化集划分：训练集 96 张图片，测试集 24 张图片。测试集仅用于模型测试，训练、调参和特征选择阶段不得使用测试集标签或 EEG 片段。



### 3.3 事件标记


| 事件代码 | 事件名称                 | 含义                            |
| ---- | -------------------- | ----------------------------- |
| 101  | `session_start`      | 实验 session 开始                 |
| 102  | `session_end`        | 实验 session 结束                 |
| 110  | `baseline_start`     | 静息基线开始                        |
| 111  | `baseline_end`       | 静息基线结束                        |
| 120  | `block_start`        | block 开始                      |
| 121  | `block_end`          | block 结束                      |
| 130  | `fixation_on`        | 注视十字出现                        |
| 131  | `fixation_off`       | 注视十字结束                        |
| 132  | `image_on`           | 图片开始呈现                        |
| 133  | `image_off`          | 图片呈现结束                        |
| 134  | `blank_on`           | 空屏缓冲开始                        |
| 135  | `blank_off`          | 空屏缓冲结束                        |
| 136  | `rating_on`          | 行为评分界面出现（范式一为同屏评分；范式二为评分阶段开始） |
| 137  | `rating_off`         | 行为评分提交完成或超时跳转                 |
| 138  | `iti_on`             | trial 间隔开始                    |
| 139  | `iti_off`            | trial 间隔结束                    |
| 140  | `trial_start`        | 单个 trial 开始                   |
| 141  | `trial_end`          | 单个 trial 结束                   |
| 142  | `attention_task_on`  | 随机注意任务出现                      |
| 143  | `attention_response` | 随机注意任务按键响应                    |
| 144  | `rating_item_on`     | 范式二：单个评分题目出现                  |
| 145  | `rating_item_off`    | 范式二：单个评分题目结束（作答或超时）           |




### 3.4 数据输出

EEG 与行为数据分别保存，典型目录结构如下：

```
records_storage/
  {subject_id}/
    {task_mode}_{session_id}/
      {timestamp}/
        events.json              # 事件标记与时间戳
        metadata.json            # session 元数据
        behavioral_ratings.csv   # Block 1 主观评分
        trial_log.csv            # 全部 trial 时序、重复次数和注意力任务记录
        image_playlist.json      # 当前 session 的图片呈现顺序
        # EEG 连续数据（格式依采集设备而定）
```

`metadata.json` 需记录当前 session 的 `task_mode`（`image_a` 或 `image_b`），以便区分图片范式一与图片范式二。

## 4. 单 Trial 流程

正式 trial 分为两类：类型 A 为首次呈现与打分，仅在 Block 1 出现；类型 B 为纯脑电降噪试次，在 Block 2-5 出现。类型 A 的评分交互由当前 session 所选图片范式决定。

### 类型 A：首次呈现与打分（共用前半段）


| 阶段   | 时长                      | 说明                               |
| ---- | ----------------------- | -------------------------------- |
| 注视点  | 500-800 ms，均值 650 ms    | 屏幕中央呈现注视点，随机抖动用于降低时间预期。          |
| 图片呈现 | 1000-1500 ms，均值 1250 ms | 呈现当前情绪图片；要求被试保持注视并尽量不眨眼。         |
| 空屏缓冲 | 500 ms                  | 图片结束后呈现空屏，作为刺激与评分阶段之间的过渡。        |
| 主观评分 | 由范式决定                   | 见下方图片范式一 / 图片范式二。                |
| ITI  | 1000-1500 ms，均值 1250 ms | trial 间隔，用于进入下一个 trial 前的短暂基线回落。 |




#### 图片范式一（`image_a` / `tasks/task_image_a.py`）

- 对应任务文件：`tasks/task_image_a.py`。
- 愉悦度、唤醒度、视觉感受三个 rating 题目在同一屏显示。
- 被试用鼠标点击完成各题目选择。
- 评分完成后，通过鼠标点击进入下一屏（ITI）；不以自动超时作为主推进方式。
- 该范式强调同屏整合判断与明确确认，减少分屏切换带来的额外眼动与按键负荷。



#### 图片范式二（`image_b` / `tasks/task_image_b.py`）

- 对应任务文件：`tasks/task_image_b.py`。
- 连续呈现 3 个 rating 题目，每屏只显示 1 题，顺序一般为愉悦度 → 唤醒度 → 视觉感受。
- 每题限时 2-3 s；超时未作答则立即跳到下一屏，要求被试快速反应。
- 操作方式仍为鼠标选择选项；后续可通过键盘 `F` / `J` 调节选项在屏幕上的位置布局，以降低固定位置偏好或运动准备偏差。
- 该范式强调单题限时与快速反应，适合采集更接近即时偏好的行为反应。



### 类型 B：纯脑电降噪试次


| 阶段    | 时长                      | 说明                                    |
| ----- | ----------------------- | ------------------------------------- |
| 注视点   | 500-800 ms，均值 650 ms    | 屏幕中央呈现注视点，随机抖动用于降低时间预期。               |
| 图片呈现  | 1000-1500 ms，均值 1250 ms | 呈现当前情绪图片；要求被试保持注视并尽量不眨眼。              |
| 注意力任务 | 约 200 ms / trial 折算     | 10% 概率触发简单按键任务，例如判断图片中是否有人影；无任务则直接跳过。 |
| ITI   | 800-1200 ms，均值 1000 ms  | trial 间隔，用于进入下一个 trial 前的短暂基线回落。      |


练习 trial 应覆盖当前所选范式的类型 A 评分操作，以及类型 B 的注意力任务操作。练习图片不进入正式 120 张刺激集合，也不进入训练集或测试集。

## 5. 实验说明首页

实验说明首页需要向被试说明：

- 本任务需要观看一系列情绪图片，并在首次观看每张图片后根据真实感受完成主观评分。
- 评分没有正确或错误答案，请根据即时主观体验作答。
- 每个题目采用 1-5 分量表。
- 当前 session 采用图片范式一或图片范式二之一：范式一为同屏 rating、评分后鼠标点击进入下一屏；范式二为连续呈现 3 个 rating、每题限时 2-3 s 并要求快速反应。
- 图片呈现期间请注视屏幕中央，尽量避免眨眼、说话、吞咽和大幅度动作。
- 部分重复观看 trial 会出现简单注意力按键任务，请按要求尽快准确作答。
- 练习 trial 用于熟悉流程，练习数据不纳入正式分析。
- 实验期间将同步采集 EEG，被试需佩戴脑电采集设备并尽量保持身体稳定。



## 6. 评分题目

图片范式一与图片范式二共用相同的 3 个评分维度，仅呈现与提交方式不同：

1. Valence：愉悦度
2. Arousal：唤醒度
3. Interest：兴趣度
4. Visual Preference：视觉感受 / 视觉偏好

所有维度均使用 5 点 Likert 量表，1 表示该维度的低端，5 表示该维度的高端。图片范式一将三题同屏呈现，评分后通过鼠标点击进入下一屏；图片范式二将三题连续呈现，每题限时 2-3 s，超时即跳到下一屏。

### 6.1 Valence / 愉悦度


| 分值  | 含义                   | 解释                             |
| --- | -------------------- | ------------------------------ |
| 1   | Very negative / 非常消极 | 观看图片时感受到非常消极的情绪，例如悲伤、愤怒、恐惧或厌恶。 |
| 2   | Negative / 消极        | 观看图片时感受到一定消极情绪，但强度低于 1 分。      |
| 3   | Neutral / 中性         | 观看图片时没有明显的积极或消极情绪。             |
| 4   | Positive / 积极        | 观看图片时感受到一定积极情绪，但强度低于 5 分。      |
| 5   | Very positive / 非常积极 | 观看图片时感受到非常积极的情绪，例如快乐、愉悦或温暖。    |




### 6.2 Arousal / 唤醒度


| 分值  | 含义                  | 解释                      |
| --- | ------------------- | ----------------------- |
| 1   | Very calm / 非常平静    | 观看图片时感到非常平静，身体和情绪反应都很弱。 |
| 2   | Calm / 平静           | 观看图片时感到较为平静，但程度低于 1 分。  |
| 3   | Neutral / 中等        | 有一定反应，但没有明显平静或兴奋。       |
| 4   | Excited / 兴奋        | 观看图片时感到明显被激活，情绪或身体反应增强。 |
| 5   | Very excited / 非常兴奋 | 观看图片时感到高度唤醒或强烈刺激。       |


### 6.3 Interest / 兴趣度


| 分值  | 含义                         | 解释                  |
| --- | -------------------------- | ------------------- |
| 1   | Very uninterested / 非常不感兴趣 | 认为视频极其无趣、乏味。        |
| 2   | Uninterested / 不感兴趣        | 认为视频有些无趣，但程度低于 1 分。 |
| 3   | Neutral / 中性               | 认为视频有中等程度的趣味性。      |
| 4   | Interested / 感兴趣           | 认为视频很有趣、很吸引人。       |
| 5   | Very interested / 非常感兴趣    | 认为视频极其有趣、非常吸引人。     |


### 6.4 Visual Preference / 视觉感受


| 分值  | 含义                    | 解释                        |
| --- | --------------------- | ------------------------- |
| 1   | Very disliked / 非常不喜欢 | 非常不喜欢这张图片的视觉呈现，观看体验很差。    |
| 2   | Disliked / 不喜欢        | 对这张图片的视觉呈现不太喜欢，但程度低于 1 分。 |
| 3   | Neutral / 中性          | 对这张图片的视觉呈现没有明显喜欢或不喜欢。     |
| 4   | Liked / 喜欢            | 比较喜欢这张图片的视觉呈现，观看体验较好。     |
| 5   | Very liked / 非常喜欢     | 非常喜欢这张图片的视觉呈现，观看体验很好。     |




## 7. 图片材料与播放列表

图片从 `visual-video-task/image_library` 中读取。每张图片需要在图片库清单中记录以下信息：

- `id`：图片唯一编号。
- `file`：图片文件相对路径。
- `emotion_category`：图片情绪类别，取值为 `positive`、`neutral`、`negative`。
- `has_person`：可选，是否包含人影，用于注意力任务。

建议 `manifest.json` 中的单个图片条目格式如下：

```json
{
  "id": "img_001",
  "file": "img_001.jpg",
  "emotion_category": "positive",
  "has_person": true
}
```



## 8. 随机化与平衡规则

每次实验开始时根据 `image_library` 随机生成一次 `image_playlist`，该播放列表在当前被试当前 session 内保持固定，并随行为数据一起保存，便于复现实验顺序。

随机化需要满足以下规则：

- 正式实验共 525 个 trial，由 105 张图片各重复 5 次构成。
- 刺激材料情绪类别平衡：共7个类别，每类15个视频。
- Block 1 包含全部 105 张图片的首次呈现与评分。
- Block 2-5 各包含同一批 105 张图片的 1 次重复呈现，每个 block 内重新乱序。
- 每个 block 内应平衡情绪类别顺序，避免同一情绪类别连续出现过多次；如无法完全避免，应至少记录该随机序列并允许后续检查。
- 注意力任务在 Block 2-5 中以约 10% 概率触发，并应在情绪类别和训练 / 测试划分上尽量均衡。
- 若某个情绪类别可用图片数量不足，应在实验开始前报错，而不是在实验中降级为不平衡播放列表。



## 9. 正式实验流程

正式实验开始后执行以下流程：

1. 连接并校验 EEG 采集设备，启动连续采集并发送 `session_start` 标记。
2. （可选）采集静息基线 EEG，发送 `baseline_start` / `baseline_end` 标记。
3. 读取 `image_library` 并校验图片文件是否存在。
4. 校验图片情绪类别是否完整，且每张图片只有一个明确的 `emotion_category`。
5. 按 8:2 生成或读取训练集 / 测试集划分：训练集 96 张，测试集 24 张，并确保测试集仅用于模型测试。
6. 根据配置选择图片范式一（`image_a`）或图片范式二（`image_b`），并写入 `metadata.json`。
7. 根据 105 张图片生成 525-trial `image_playlist`：Block 1 首次呈现与评分，Block 2-5 为乱序重复呈现。
8. Block 1 对每个 trial 执行：注视点、图片呈现、空屏缓冲、当前范式评分、ITI；各阶段边界发送对应事件标记。
9. Block 2-5 对每个 trial 执行：注视点、图片呈现、可选注意力任务、ITI；各阶段边界发送对应事件标记。
10. 每个 block 开始 / 结束时发送 `block_start` / `block_end` 标记；Block 1 结束后安排 2-3 分钟闭眼休息，Block 2-4 结束后安排约 30 秒短休。
11. 每个 trial 保存范式标识、图片编号、情绪类别、训练 / 测试划分、block 编号、trial 编号、重复次数、各阶段时间戳、评分结果、评分是否超时、注意力任务与按键反应。
12. 实验结束后发送 `session_end` 标记，停止 EEG 采集，并保存完整行为评分表、事件日志、播放列表和 EEG 数据。

正式实验预计时长如下。Block 2-5 在两种范式下相同；Block 1 时长随评分范式变化，整体仍宜控制在约 45 分钟量级：


| 实验阶段         | 试次内容               | Trial 数量 | 单个 Trial 均时               | 纯耗时预估     | 建议操作              |
| ------------ | ------------------ | -------- | ------------------------- | --------- | ----------------- |
| Block 1（范式一） | 全量图片，首次观看 + 同屏评分   | 105      | 约 8-10 秒（取决于确认点击节奏）       | ~16-20 分钟 | 完成后安排 2-3 分钟闭眼休息。 |
| Block 1（范式二） | 全量图片，首次观看 + 连续限时评分 | 105      | 约 9.15-12.15 秒（三题各 2-3 s） | ~18-24 分钟 | 完成后安排 2-3 分钟闭眼休息。 |
| Block 2      | 乱序重排，仅被动注视 + 按键    | 105      | 3.10 秒                    | ~6.2 分钟   | 完成后短休 30 秒。       |
| Block 3      | 乱序重排，仅被动注视 + 按键    | 105      | 3.10 秒                    | ~6.2 分钟   | 完成后短休 30 秒。       |
| Block 4      | 乱序重排，仅被动注视 + 按键    | 105      | 3.10 秒                    | ~6.2 分钟   | 完成后短休 30 秒。       |
| Block 5      | 乱序重排，仅被动注视 + 按键    | 105      | 3.10 秒                    | ~6.2 分钟   | 实验结束，清洗电极。        |




## 10. 数据记录字段

每个 trial 建议至少记录以下字段：


| 字段                                                 | 说明                                   |
| -------------------------------------------------- | ------------------------------------ |
| `subject_id`                                       | 被试编号。                                |
| `session_id`                                       | session 编号。                          |
| `task_mode`                                        | 图片范式：`image_a`（范式一）或 `image_b`（范式二）。 |
| `block_idx`                                        | block 编号，取值 1-5。                     |
| `trial_idx`                                        | 当前 trial 编号，取值 1-600。                |
| `block_trial_idx`                                  | 当前 block 内 trial 编号，取值 1-120。        |
| `image_id`                                         | 图片唯一编号。                              |
| `image_file`                                       | 图片文件路径。                              |
| `emotion_category`                                 | 图片情绪类别：正性、中性或负性。                     |
| `split`                                            | 建模划分：训练集或测试集。                        |
| `repeat_idx`                                       | 当前图片第几次呈现，取值 1-5。                    |
| `trial_type`                                       | `rating` 或 `eeg_denoise`。            |

## 11. 实现注意事项

- 当前目标参数为：105 张情绪图片，共7类情绪图片，每类图片各15张；每张图片重复呈现 5 次，总计 525 个 trial。
- 两种评分方式定义为两个图片范式，分别对应 `tasks/task_image_a.py`（图片范式一，`task_mode=image_a`）与 `tasks/task_image_b.py`（图片范式二，`task_mode=image_b`）；同一 session 只启用一种范式。
- Block 1 共用前半段时序：注视点 500-800 ms、图片 1000-1500 ms、空屏 500 ms、ITI 1000-1500 ms；评分阶段按范式分支。
- 图片范式一：三个 rating 同屏显示，评分后通过鼠标点击进入下一屏；需记录确认点击事件。
- 图片范式二：连续呈现 3 个 rating，每题限时 2-3 s，超时即跳到下一屏，要求快速反应；操作方式仍为鼠标选择，后续可通过键盘 `F` / `J` 调节选项位置。需记录每题 onset/offset、反应时与是否超时。
- Block 2-5 采用类型 B 流程：注视点 500-800 ms、图片 1000-1500 ms、10% 概率注意力任务、ITI 800-1200 ms；两种范式共用。
- 图片库需要补充 `emotion_category`、`split` 和注意力任务所需元数据，否则无法实现情绪类别平衡、泛化集隔离和注意力任务校验。
- 训练集 / 测试集划分必须以图片为单位，而不是以 trial 为单位，避免同一图片的重复 trial 同时出现在训练和测试中。
- 注意力任务应仅用于维持被试注视和参与度，不应频繁出现，以免改变被动观看任务性质。
- 行为任务与 EEG 采集必须同步启动与结束；各 trial 阶段切换时须发送对应事件标记，保证后续可对齐分析。
- EEG 设备类型与连接参数在 `config.yaml` 中配置（如 `device_type`、`sfreq`、`task_mode`）；正式采集前需完成阻抗检查与信号质量确认。



## 12. 程序首页实验说明

以下文字可用于实验程序首页，面向被试展示。说明正文中评分操作部分需按当前 session 所选范式切换。

### 实验说明

欢迎参加本次图片观看与主观评分实验。

在本实验中，你将佩戴脑电采集设备（电极帽），同时观看一系列情绪图片。程序会同步记录你的脑电信号，用于科研分析。请在整个实验过程中尽量保持头部和身体稳定，减少说话、吞咽和大幅度动作。

每张图片第一次出现后，请根据你观看图片时的真实感受，对图片进行愉悦度、唤醒度和视觉感受评分。评分没有正确或错误答案，请按照你的第一感受作答。后续部分图片会重复出现，重复观看阶段通常不需要评分，只需保持注视；少数 trial 会出现简单按键任务，请按提示尽快准确作答。

### 实验流程

实验开始后，每个 trial 会按照以下顺序进行：

1. 屏幕中央先出现一个注视点，请注视屏幕中央并做好观看准备。
2. 随后呈现一张图片。图片呈现时间较短，请保持注视并尽量不要眨眼。
3. 如果这是该图片第一次出现，图片结束后会出现短暂空屏，然后进入评分页面。
4. 请按当前范式完成愉悦度、唤醒度和视觉感受评分（见下方评分方式）。
5. 如果这是重复观看阶段，图片结束后通常会直接进入短暂间隔；少数 trial 会出现注意力按键任务。
6. 间隔结束后，程序会自动进入下一张图片。

正式实验前会先进行练习 trial。练习 trial 的目的是帮助你熟悉当前范式的评分方式和注意力按键任务，练习数据不会纳入正式分析。

### 评分方式

每个题目使用 1-5 分评分。一般来说，1 表示该维度非常低或非常弱，3 表示中等或中性，5 表示该维度非常高或非常强。

#### 图片范式一（同屏评分）

三个评分题目会显示在同一页面。请用鼠标完成选择；评分完成后，再通过鼠标点击进入下一屏。

#### 图片范式二（连续限时评分）

三个评分题目会连续出现，每屏只显示一题。请尽快用鼠标选择；每题限时约 2-3 秒，超时会立即跳到下一屏。后续可通过键盘 `F` / `J` 调节选项位置，请按屏幕提示操作。

你需要完成以下 3 个维度的评分：

1. 愉悦度 Valence
  请评价你观看图片时的情绪是偏消极还是偏积极。
  - 1 分：非常消极，例如感到悲伤、愤怒、恐惧、厌恶或不舒服。
  - 3 分：中性，没有明显积极或消极情绪。
  - 5 分：非常积极，例如感到开心、愉悦、温暖或享受。
2. 唤醒度 Arousal
  请评价你观看图片时身体和情绪被激活的程度，也就是你有多平静或多兴奋。
  - 1 分：非常平静，心情和身体状态都很放松。
  - 3 分：中等，没有明显平静或兴奋。
  - 5 分：非常兴奋或被强烈刺激，感觉情绪或身体反应明显增强。
3. 视觉感受 Visual Preference
  请评价你对这张图片视觉呈现的主观喜欢程度。
  - 1 分：非常不喜欢，观看体验较差。
  - 3 分：中性，没有明显喜欢或不喜欢。
  - 5 分：非常喜欢，观看体验很好。

请注意，每个评分都只针对刚刚观看的那张图片。即使连续观看多张图片，也请独立评价当前图片，不需要和前面的图片进行比较。
